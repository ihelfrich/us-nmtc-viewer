# NMTC Viewer — browser-based first look

A single-page interactive presentation of the NMTC data bundled in this
repo. Built for Katia but works as a general explainer for the program.

## What's inside

A dark-themed scrolling presentation with:

1. **Hero + What is NMTC** — plain-English explainer plus an SVG money-flow diagram
   (Investor → CDE → QALICB, with the 39% credit flowing back).
2. **Headline KPI cards** — read from `data/headline.js` at runtime.
3. **The map** — CesiumJS globe with Esri World Imagery + 8,019 projects plotted as
   colored dots at their 2020-census-tract centroid. Interactive:
   - two-handle year-range slider (2001–2022)
   - metro / non-metro toggles
   - QALICB-type filter (RE / NRE / SPE / CDE)
   - color-by switch: metro status / leverage ratio / QALICB type
   - hover any dot for a detail card; click to pin it
   - live count + $-total updates as you filter
4. **Time series** — embeds the two annual-deployment PNGs.
5. **Leverage gap** — embeds the three distribution/composition PNGs.
6. **CDE heterogeneity table** — top-20 CDEs sorted by non-metro share, with
   gradient bars showing their metro/non-metro mix.
7. **Econometrics** — four estimation specs rendered with KaTeX:
   - leverage definition + mobilization ratio
   - metro × CDE fixed-effects regression
   - LIC-eligibility regression discontinuity
   - 20% mandate bunching test (Chetty/Kleven-style excess-mass integral)
8. **Where next** — links back to the brief.

## How to open it

**Option A — local web server (recommended):**

```bash
cd us_nmtc/
python3 -m http.server 8765
# then open http://localhost:8765/viewer/
```

Any simple static-file server works. Python's built-in is zero-setup.

**Option B — double-click `index.html`:**

The viewer is deliberately built without `fetch()` — data is served via
`<script src>` tags so it works from `file://` too. Just double-click
`index.html` and it should open in your default browser. (Some browsers
lock down `file://` more aggressively than others; if anything misbehaves,
fall back to Option A.)

## What it depends on

CDN-loaded at runtime (requires internet on first view):

- **CesiumJS 1.118** — the globe
- **Esri World Imagery** (tokenless) — the base layer
- **KaTeX 0.16.11** — the equations
- **Google Fonts** — Inter + Newsreader + IBM Plex Mono

All of these are public, unauthenticated CDN endpoints.

## How to rebuild the data bundle

If you refresh `data/processed/` (e.g. after editing `scripts/describe_nmtc.py`)
you need to regenerate the viewer's JS data files too:

```bash
python3 scripts/build_viewer_data.py
```

That script:
1. Downloads the US Census 2020 tract-level gazetteer (cached in `data/cache/`)
2. Merges tract FIPS → lat/lon for every project
3. Writes four JS data modules into `viewer/data/`:
   - `projects.js`   — 8,019 records (≈ 1.6 MB) with lat/lon/QLICI/metro/type/leverage
   - `states.js`     — 55-state rollup
   - `headline.js`   — the headline KPI numbers
   - `top_cdes.js`   — top-20 CDE leaderboard

## Known quirks

- **Offline use.** Cesium + KaTeX + fonts are fetched from CDNs. Works great
  with internet; will degrade without it (the page will still load, but
  the map won't render imagery and equations will show as raw `$...$`).
- **Auto-render scope.** KaTeX's auto-render is explicitly scoped to the
  `#econ` section so that literal `$5M` / `$1M` elsewhere in the prose
  are left as-is rather than being consumed as math delimiters.
- **5 projects dropped.** Five of the 8,024 projects have tract FIPS
  that don't appear in the 2020 gazetteer (likely renumbered tracts).
  They're omitted from the map but present in the CSVs.
